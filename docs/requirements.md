# 要件定義書

## プロジェクト目的

ユーザーの生活習慣パターンとカレンダー予定を統合し、移動時間を考慮した最適な1日のスケジュールを自動生成する。リアルタイムで進捗を追跡し、遅延時には自動調整を行うことで、効率的な時間管理を支援する。

## ユーザーペルソナ

- **ターゲットユーザー**: 予定が多く、時間管理を効率化したい個人
- **利用シーン**:
  - 朝起きてその日のスケジュールを確認
  - 1日を通してイベント完了をチェック
  - 予定が遅延した際の影響を即座に把握
  - 移動時間を考慮した行動計画の立案

## 機能要件

### 1. 生活習慣パターン管理

#### 1.1 パターンCRUD
- [ ] 複数の生活習慣パターンを作成・編集・削除できる
- [ ] パターンごとに名前を設定できる（例：平日パターン、休日パターン）
- [ ] パターンごとに適用ルールを設定できる

#### 1.2 適用ルール設定
- [ ] 曜日指定（複数選択可）
- [ ] キーワードマッチング（カレンダー予定のタイトルに含まれるキーワード）
- [ ] デフォルトパターンの指定
- [ ] 優先度の設定（複数ルールに該当する場合の判定）

#### 1.3 習慣項目のカスタマイズ
- [ ] 固定項目：起床時間、就寝時間
- [ ] カスタム項目：朝食、昼食、夕食、運動、勉強、休憩、入浴など
- [ ] 各項目の設定内容：
  - 名前
  - 開始時刻
  - 所要時間
  - デフォルト場所（任意）
  - アイコン（任意）
  - 色（任意）
  - 時間調整可否フラグ（柔軟性）
  - 優先度

### 2. カレンダー連携

#### 2.1 .ics ファイルインポート（MVP ✅ 実装済み）
- [x] .ics（iCalendar RFC 5545）ファイルをアップロードしてイベントをインポート
- [x] 対応アプリ：TimeTree、Google Calendar、Apple Calendar など .ics エクスポート対応アプリ全般
- [x] 差分更新（追加・更新・削除）による効率的な同期
- [x] インポート済みイベントは設定画面で確認・削除可能
- [x] 取得情報：
  - イベントタイトル
  - 開始時刻・終了時刻
  - 場所情報
  - 説明文
  - 終日フラグ

#### 2.2 場所の自動マッチング（未実装）
- [x] カレンダー予定の場所情報を場所マスタとマッチング
- [x] 別名（エイリアス）による柔軟なマッチング
- [x] マッチしない場合の処理（前回の場所を継承）

#### 2.3 Google カレンダーとのリアルタイム同期（将来：v2）
- [ ] Google Calendar REST API v3 を使用
- [ ] OAuth 2.0（Authorization Code Flow）による認証
- [ ] events.list API でイベントを自動取得
- [ ] syncToken による差分同期（初回全件取得→以降更新分のみ）
- [ ] 手動同期ボタン + 設定した間隔での自動同期
- [ ] セキュリティ：OAuth フローは Next.js Route Handler で処理、Client Secret は Vercel 環境変数に保管

### 3. 場所・移動時間管理

#### 3.1 場所マスタ管理
- [ ] 場所の登録・編集・削除
- [ ] 場所の設定項目：
  - 名前（例：自宅、会社、ジム）
  - 別名（エイリアス）の複数登録
  - 住所（任意・将来的にAPI連携用）
- [ ] デフォルト開始位置の設定（通常は自宅）

#### 3.2 移動ルート管理
- [ ] 移動ルートの登録・編集・削除
- [ ] ルートの設定項目：
  - 出発地（場所マスタから選択）
  - 目的地（場所マスタから選択）
  - 移動手段（徒歩、車、電車、バス、自転車、その他）
  - 所要時間（分）
  - デフォルト手段フラグ
- [ ] 同じ区間で複数の移動手段を登録可能
- [ ] 移動手段の切り替え機能

#### 3.3 移動時間の自動計算
- [ ] 場所が変わるイベント間に移動時間を自動挿入
- [ ] デフォルト移動手段の使用
- [ ] 移動時間が足りない場合の警告

### 4. スケジュール自動生成

#### 4.1 スケジュール生成ロジック
- [ ] 生活習慣パターンの選択（ルールベース）
- [ ] 習慣項目とカレンダー予定のマージ
- [ ] 時系列順のソート
- [ ] 現在地追跡による移動イベントの挿入
- [ ] 空き時間の計算・表示

#### 4.2 現在地追跡
- [ ] 1日の開始位置（デフォルト：自宅）
- [ ] イベント実行後の現在地更新
- [ ] 場所が未設定のイベントは現在地のまま

#### 4.3 衝突・重複の処理
- [ ] カレンダー予定と習慣項目が重複する場合、カレンダー予定を優先
- [ ] 習慣項目の自動スキップ
- [ ] ユーザーへの通知

### 5. リアルタイム進捗追跡

#### 5.1 イベント状態管理
- [ ] イベントの状態：pending、active、completed、skipped
- [ ] イベント完了チェック機能
- [ ] 現在のイベント・次のイベントの表示
- [ ] 次のイベントまでのカウントダウン（リアルタイム更新）

#### 5.2 遅延検出
- [ ] イベント終了時刻と実際の完了時刻の比較
- [ ] 遅延時間の記録
- [ ] 遅延の累積計算

#### 5.3 自動調整
- [ ] 柔軟なイベントの時間を後ろにずらす
- [ ] カレンダー予定（固定）には影響させない
- [ ] 移動時間の最適化（より速い手段の提案）
- [ ] 調整不可能な場合の警告

#### 5.4 通知機能（将来対応）
- [ ] 次のイベントの通知
- [ ] 遅延の警告
- [ ] 移動開始のリマインダー

## 非機能要件

### パフォーマンス
- [ ] スケジュール生成は1秒以内
- [ ] リアルタイム更新は遅延なく（1秒以内）
- [ ] ページ読み込みは2秒以内

### セキュリティ
- [ ] Apple Calendar認証情報の安全な保管
- [ ] XSS、CSRF対策
- [ ] 機密情報のログ出力禁止

### 可用性
- [ ] オフライン時も基本機能が動作（LocalStorageベース）
- [ ] エラー時のフォールバック

### 保守性
- [ ] TypeScriptによる型安全性
- [ ] コンポーネントの再利用性
- [ ] ドキュメントの整備

### 互換性
- [ ] モダンブラウザ対応（Chrome、Safari、Firefox、Edge）
- [ ] レスポンシブ対応（PC、タブレット、スマートフォン）

## 制約条件

### 技術的制約
- Next.js App Routerを使用
- Vercelにデプロイ
- MVPではLocalStorageのみ使用（クラウド非対応）

### スコープ外（v1では対応しない）
- 複数ユーザー対応
- チーム共有機能
- 外部API（Google Maps等）による移動時間計算
- プッシュ通知
- ダークモード
- 多言語対応
